Q. var, let, const의 차이에 대해 알고있는가?

```
var는 let과 const가 등장하기 이전에 사용되던 변수 선언 방법입니다.
var는 함수 범위를 가지기 떄문에 함수 내에서 선언된 변수는 해당 함수 전체에서 접근 가능합니다.
그렇기 때문에 해당 함수내에서 선언된 if문과 같은 블록에서 선언된 변수일지라도
해당 함수에서 접근이 가능하게 됩니다. 이는 코드의 복잡성이 증가하고 예상치 못한 변수 값변경이 발생할 수 있는 위험이 있습니다.

하지만 let과 const는 ES6 이후 도입된 변수 선언 방법으로, 블록 범위를 가지고 있습니다.
이는 변수의 유효 범위가 해당 블록 내로 제한되어 블록 외부에서는 접근할 수 없습니다. 그렇기 때문에 if문과 같은 블록에서 선언된 변수는 해당 블록내에서만 접근이 가능하므로 코드의 가독성과 유지보수성을 높여줍니다.

let은 재할당이 가능한 변수를 선언할 때 사용되며, 값의 변경이 필요한 변수에 적합합니다.
한편, const는 상수(constant)를 선언할 때 사용되며, 선언과 동시에 반드시 초기 값을 할당해야 합니다.
그리고 한 번 할당된 값은 변경할 수 없습니다.

따라서, 일반적인 규칙으로는 var 키워드를 사용하지 않고, 값이 변경될 수 있는 변수에는 let을 사용하고, 한 번 할당된 값이 변경되지 않는 상수에는 const를 사용하는 것이 좋습니다.
이를 통해 변수의 스코프를 명확히하고 의도하지 않은 변수 값 변경이나 중복 선언과 같은 문제를 방지할 수 있습니다.
```

Q. var를 사용해야 하는 경우가 있을까요?

```
ES6 이전의 브라우저 환경에서 작업해야 하는 경우에 호환성문제로 인해 var를 사용할 수 있을것 같긴 하지만 그런 환경이 아니라면 var의 사용은 지양하는 것이 좋다고 생각합니다.
```

Q. let과 const의 블록 범위에 대해 좀 더 구체적인 예시를 들어볼 수 있을까요?

```
let 과 const는 블록 범위를 가지고 있습니다. 이는 변수의 유효 범위가 해당 블록내로 제한되어 블록 외부에서는 접근할 수 없다는 의미입니다.
예를들어 어떤 함수안에 if문이 있다고 가정해보겠습니다. 그리고 if문 안에서 var, let, const로 각각 선언한 변수가 있다고 가정하겠습니다. 이때 if문 안에서 각변수들에 접근이 가능하지만 if문바깥인 함수의 공간에서 let이나 const로 선언한 변수에는 접근이 불가능하지만 var로 선언한 변수는 접근이 가능합니다. 이는 앞서 말씀드린 var는 함수범위의 스코프를 가지기때문이고 let과 const는 블록범위 스코프를 가지기 때문에 일어나는 특징이라고 볼 수 있습니다.
```

Q. const로 선언된 상수의 값은 변경할 수 없다고 하셨는데, 그렇다면 객체나 배열과 같은 복합 데이터 타입을 const로 선언하면 어떻게 되나요?

```
객체나 배열과 같은 복합 데이터 타입은 const로 선언하더라도 해당 객체나 배열의 내부 내용은 변경될 수 있습니다. 다만 선언된 객체나 배열을 다른 객체나 배열로 다시 할당하는 것은 불가합니다. 예를들어 const로 선언한 person이라는 객체안에 name이 정수현이고 age가 29살이라는 정보가 들어있다고 가정해보겠습니다. 이때 person.name = 김강철 과 같이 객체의 프로퍼티를 바꾸는 것은 가능하나 person 이라는 변수 자체에 새로운 객체를 할당하는 것은 불가능합니다. 배열도 마찬가지로 이미 선언된 배열에 값을 추가하거나 삭제하는 것은 가능하나 배열을 새로운 배열로 할당하는 것은 불가능합니다. 따라서 const로 선언된 객체나 배열은 내부 내용을 변경할 수 있지만, 다른 객체나 배열로 재할당하는 것은 허용되지 않습니다.
```

Q. let과 const의 호이스팅 동작에 대해 알고 계신가요?

```
let과 const는 호이스팅이 발생합니다. 호이스팅은 자바스크립트 엔진이 코드를 해석할 때, 변수 선언부 또는 함수 선언부를 먼저 처리하여 해당 스코프의 최상단으로 옮기는 동작입니다. 즉, 변수 또는 함수가 선언되기 이전에도 사용할 수 있다는 것을 의미합니다.

하지만 let과 const는 호이스팅이 되지만 초기화가 되지 않습니다. 변수 선언부만 호이스팅되고, 실제 변수에 값을 할당하는 초기화 단계는 선언문에 도달했을 때 실행됩니다. 이러한 이유로 선언 이전에 변수에 접근하려고 하면 "ReferenceError"가 발생합니다.

따라서 let과 const 변수를 사용할 때에는 변수를 선언한 이후에 사용하는 것이 좋습니다. 이렇게 변수를 사용하기 전에 선언하면 호이스팅과 관련된 문제를 방지할 수 있고, 코드의 가독성과 유지보수성을 향상시킬 수 있습니다.
```
