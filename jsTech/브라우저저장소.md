Q. 브라우저 저장소에 대해 설명해주세요

```
브라우저 저장소에는 웹 스토리지와 쿠키가 있습니다. 우선 쿠키는 서버와 클라이언트 간의 지속적인 데이터 교환을 위해 만들어진 저장소입니다.
주로 사용자 인증, 세션 관리, 개인화 등에 활용이 됩니다. 쿠키는 4kb의 작은 용량 제한을 가지고 있고 개수의 제한이 있습니다. 따라서 범용성있게 사용하기에는 조금 한계가 있었습니다.
이러한 단점을 보완하기 위해 등장한 것이 웹 스토리지입니다.

웹 스토리지는 데이터를 서버로 전송하지 않고 오직 클라이언트 쪽에서만 주고받으며 저장공간이 5mb로 쿠키보다 크다는 특징이 있습니다.

이러한 웹스토리지에는 로컬스토리지와 세션스토리지가 존재하는데 우선 로컬스토리지는 브라우저를 종료하여도 데이터가 유지됩니다.
따라서 한 번 저장된 정보의 경우 명시적으로 삭제 해주지 않는 이상 데이터가 끝까지 남아있게 됩니다.
그렇기 때문에 다크모드 설정이나 로그인 정보와 같은 정보들을 저장할 수 있습니다.

세션 스토리지의 경우 로컬스토리지와 다르게 브라우저를 종료하면 데이터가 삭제가 됩니다. 또한 브라우저를 이동하더라도 정보가 사라지게 됩니다.
그렇기 때문에 주로 장바구니를 구현할 때 사용하는 저장소입니다.
```
