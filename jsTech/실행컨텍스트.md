Q. 실행 컨텍스트에 대해 설명해주세요

```
실행 컨텍스트는 코드가 실행되는 환경을 말하며, 모든 JS 코드는 실행컨텍스트 내부에서 실행됩니다.
실행 컨텍스트에는 전역 컨텍스트, 함수 컨텍스트, eval 함수 컨텍스트 이렇게 세가지 종류가 있는데 eval 함수 컨텍스트는 잘 사용되지 않아
나머지 두개의 컨텍스트에 대해 설명해드리겠습니다.

먼저 전역 컨텍스트는 script 요소를 맨 처음 만나는 시점에 생성이 됩니다. 생성된 전역 컨텍스트는 콜스택에 가장먼저 쌓이게 됩니다.
전역 컨텍스트 안에는 전역객체, 전역변수, 내장객체와 함수, this와 같은 정보가 포함되며 전역 컨텍스트에 선언된 변수와 함수는 모든 스코프에서
접근 가능하며 애플리케이션의 실행동안 게속해서 유지가 됩니다.

그리고 함수가 호출이 될 때마다 함수 컨텍스트 역시 콜스택에 쌓이게 됩니다. 콜스택에 쌓인 컨텍스트 들은 가장위에에서부터 콜스택을
순차적으로 비워나가게 됩니다. 함수 컨텍스트안에는 변수객체, scope chain, this에 대한 정보가 생성이 됩니다.
함수가 실행이 될때 사용되는 변수들을 변수객체 안에서 찾게되고 없다면 스코프체인을 따라 올라가며 찾게되고 this의 값 또한 이때 결정이됩니다.
그렇게 함수의 실행이 마무리 되면 콜스택에 쌓여있던 해당 함수 컨텍스트는 사라지게 되고 페이지가 종료되면 전역 컨텍스트 역시 사라지게 됩니다.

실행 컨텍스트는 처음에 말씀드린 것처럼 코드가 실행되는 환경이며 모든 js 코드는 실행컨텍스트 내부에서 실행이됩니다.
따라서 실행 컨텍스트를 이해하는 것은 js의 동작 원리를 이해하는데 많은 도움이 됩니다. 또한 js엔진은 실행 컨텍스트를 통해 코드의
실행 흐름을 보장할 수 있습니다.
```
