Q. API란 무엇인가?

```
API는 일종의 계약서로 생각할 수 있습니다. 개발자는 외부 서비스나 시스템의 기능을 활용하기 위해
API를 사용하여 요청을 보내고, 응답을 받아 처리합니다. 이를 통해 다른 개발자나 서비스가 제공하는 기능을 직접 호출하여 사용할 수 있습니다.

API는 다양한 형태로 제공될 수 있는데, 웹 API의 경우 HTTP를 통해 요청과 응답을 주고받는 REST API가 흔히 사용되며, JSON 또는 XML 형식으로 데이터를 주고 받습니다.

정리하자면 API는 개발자에게 다른 시스템의 기능을 쉽게 활용할 수 있는 기회를 제공하며, 시스템 간의 기능 확장에 유용합니다.

다양한 서비스와 플랫폼이 API를 제공하고 있으며, 개발자는 이를 활용하여 자신의 애플리케이션을 더욱 풍부하게 만들 수 있습니다.

실제로 진행했던 뮤직플레이어 프로젝트에서 deezer라는 open API를 활용하여 전세계의 다양한 음악을 검색및 추가 할 수 있는 프로젝트를 진행한 바가 있습니다.
```

Q. REST API란 무엇인가?

```
REST API는 웹 서비스간의 통신을 위한 아키텍처 스타일입니다. REST는 자원 기반, HTTP 메서드 활용, 상태 없음, 캐시 가능의 네가지 특징을 가집니다.

먼저 자원 기반 특징은 각 리소스는 고유한 식별자를 가지며 URL를 통해 표현됩니다.
HTTP 메서드 활용은 HTTP 프로토콜의 메서드인 GET,POST,PUT,DELETE 를 활용하ㅕㅇ 자원에 대한 행위를 수행합니다.
상태 없음은 서버는 클라이언트의 상태를 관리하지 않으며, 각 요청은 필요한 모든 정보를 포함하고 있어야합니다.
캐시 가능은 HTTP 프로토콜의 개싱 기능을 활용하여 서버의 응답을 클라이언트가 캐시할 수 있습니다. 이를 통해 네트워크 대역폭을 절약하고
응답 시간을 단축시킬 수 있습니다.

이러한 REST 아키텍처의 특징을 엄격하게 준수한 것을 RESTful API라고 합니다. 반면에 REST API는 RESTful API와 비교하여 REST의 원칙을 덜 엄격하게 따르는
API를 일컫는 경우도 있습니다.
```
