Q. Promise에 대해 알고있는가?

```
Promise는 자바스크립트에서 비동기 처리를 다루기 위한 객체입니다. 비동기 작업이란, 요청을 보낸 후 결과를 기다리는 동안 다른 작업을 처리할 수 있는 것을 말합니다. Promise는 이러한 비동기 작업을 보다 편리하게 다룰 수 있도록 도와줍니다.

Promise는 대기(pending), 이행(fulfilled), 거부(rejected)의 세가지 상태를 가질 수 있는데, 대기 상태에서는 작업이 진행 중이며, 이행 상태에서는 작업이 성공적으로 완료되었고 결과값을 반환합니다. 거부 상태에서는 작업이 실패하였거나 오류가 발생한 상태입니다.

Promise는 then()과 catch() 메서드를 제공하여 비동기 작업의 성공 또는 실패를 처리할 수 있습니다. then()은 작업이 성공한 경우 실행되며, catch()는 작업이 실패한 경우 실행됩니다.

또한, Promise는 연속적인 비동기 작업을 처리하기 위해 체이닝(Chaining)을 지원합니다. then() 메서드를 여러 번 연결하여 비동기 작업을 순차적으로 실행할 수 있습니다. 또한, Promise.all() 메서드를 사용하여 여러 개의 Promise를 병렬로 실행하고 모든 작업이 완료될 때까지 기다릴 수도 있습니다.

요약하자면 Promise는 비동기 작업을 처리하기 위한 자바스크립트의 내장 객체로, 비동기 코드를 보다 구조적이고 가독성 있게 작성할 수 있도록 도와줍니다.
```

Q. Promise는 어떻게 비동기 작업을 보다 편리하게 도와주는가?

```
then() 메서드를 통해 비동기 작업이 완료된 후에 처리할 로직을 지정할 수 있습니다.
이를 통해 비동기 작업을 순차적으로 실행하고 결과를 처리할 수 있게 됩니다. 또한 then() 메서드를 체인으로 연결하면
여러개의 비동기 작업을 간편하게 순차적으로 처리할 수 있습니다.

또한 예외처리 부분에서도 catch() 메서드를 통해 비동기 작업중 발생한 오류를 처리할 수 있습니다.
Promise 체인에서 발생한 오류는 catch() 블록에서 캐치되어 추가적인 오류처리를 해줄 수 있습니다. 이를 통해 오류처리 로 직을 효율적으로 구성이 가능하고
코드의 예외 상황에 대한 처리가 명확해집니다.
```

Q. Callback 함수와의 차이점을 알고 있는가?

```
콜백함수는 비동기 로직의 결과값을 처리하기 위해서 콜백안에서만 처리를해야하고 콜백 밖에서는 비동기에서 온 값을 알 수가 없습니다.
하지만 프로미스의 경우 비동기에서 온 값이 프로미스 객체에 저장되기 때문에 코드의 작성이 용이해집니다.
또한 콜백함수의 경우 비동기 작업을 호출하고 작업이 완료되면 콜백함수를 호출하여 결과를 기다립니다. 비동기 작업이 중첩되는 경우에
콜백 지옥에 빠지게 되어 코드의 가독성이 떨어질 수 있습니다. 그러나 Promise에서는 then메서드를 통해 비동기적인 작업을 편리하게 해줄 수 있어서
콜백함수의 콜백 지옥을 피할 수 있고 비동기 코드를 보다 구조적이고 가독성있게 작성할 수 있도록 도와준다는 차이가 있습니다.
```
