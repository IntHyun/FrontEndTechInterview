Q. 자바스크립트의 동작 원리에 대해 설명하세요.

```
JavaScript의 동작 원리는 크게 두 가지 요소에 기반을 두고 있습니다: 실행 환경(Execution Environment)과 이벤트 루프(Event Loop)입니다.

실행 환경 (Execution Environment):

콜 스택 (Call Stack): 함수 호출과 관련된 정보를 저장하는 스택입니다. 함수가 실행되면 해당 함수의 정보가 콜 스택에 쌓이고, 실행이 완료되면 해당 정보가 스택에서 제거됩니다.
웹 API (Web APIs): 브라우저 환경에서 제공하는 비동기 작업을 처리하기 위한 API들입니다. 예를 들어, setTimeout, XMLHttpRequest, fetch 등이 웹 API에 해당합니다.
콜백 큐 (Callback Queue): 비동기 작업의 완료나 이벤트 발생과 같은 이벤트를 대기하는 큐입니다. 웹 API에서 비동기 작업이 완료되면 해당 콜백 함수가 콜백 큐로 이동됩니다.
이벤트 루프 (Event Loop): 콜 스택과 콜백 큐를 감시하며, 콜 스택이 비어있을 때 콜백 큐의 작업을 콜 스택으로 이동시킵니다.
이벤트 루프 (Event Loop):

이벤트 루프는 실행 환경에서 콜 스택과 콜백 큐를 감시하며 동작합니다.
콜 스택이 비어있을 때, 이벤트 루프는 콜백 큐에서 우선순위가 가장 높은 작업을 콜 스택으로 이동시킵니다.
이벤트 루프는 단일 스레드로 동작하며, 비동기 작업을 처리하고 응답성을 유지하기 위해 콜백 큐의 작업을 적절히 조정합니다.
JavaScript의 동작은 이벤트 루프를 통해 비동기 작업을 처리하고, 콜 스택을 통해 함수 호출을 관리합니다. 이벤트 루프는 콜 스택이 비어있을 때 콜백 큐의 작업을 콜 스택으로 이동시키는 역할을 수행하여 비동기 작업을 순차적으로 처리합니다. 이를 통해 JavaScript는 단일 스레드로 동작하면서도 비동기적인 동작을 지원하고 응답성을 유지할 수 있습니다.
```
