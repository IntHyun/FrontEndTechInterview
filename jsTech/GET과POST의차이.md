Q. GET과 POST의 차이에 대해 알려주세요

```
GET method는 서버로부터 정보를 요청하기 위해 주로 사용되는 method입니다.
GET 요쳥은 전송할 때 필요한 데이터를 Body에 담지 않고 쿼리스트링을 통해 전송합니다.
URL 끝에 ? 와 함께 이름과 값으로 쌍을 이루는 요청 파라미터를 쿼리스트링이라 하는데 이러한 특징때문에 GET 요청은
민감한 데이터를 다룰때 사용하면 적합하지 않은 메서드가 될 수 있습니다.

또한 GET 요청은 불필요한 요청을 제한하기 위해 요청이 캐시될 수 있습니다. js, css, 이미지 같은 정적 컨텐츠는 데이터의 양이 크고
변경될 일이 적어서 반복해서 동일한 요청을 보낼 필요가 없습니다.
따라서 정적 컨텐츠를 요청하고 나면 브라우저에서는 요청을 캐시해두고 동일한 요청이 발생할 때 서버로 요청을 보내지 않고 캐시된 데이터를 사용합니다.

POST 요청은 리소스를 생성/변경하고자 할때 사용되는 메서드입니다. 전송해야될 데이터를 HTTP 메세지의 Body에 담아서 전송하는데,
HTTP 메세지의 Body는 길이의 제한없이 데이터를 전송할 수 있습니다. 그래서 POST 요청은 GET 요청과 달리 대용량 데이터를 전송할 수 있습니다.
또한 데이터가 Body로 전송되고 내용이 눈에 보이지않아 GET보다 보안적인 면에서는 안전하다고 할 수 있습니다.
그러나 POST 요청도 크롬 개발자도구로 요청 내용을 확인할 수 있기에 민감한 데이터의 경우에 반드시 암호화하여 전송하도록 해야합니다.

그리고 POST로 요청을 보낼 때는 요청 헤더의 Content-Type에 요청 데이터의 타입을 표시해야 합니다.

따라서 GET과 POST의 차이를 정리하자면 GET은 데이터를 요청할때 사용하며 데이터를 body에 담지않으므로 민감한 데이터를 다루기 적합하지 않습니다. POST는 리소스를 생성하거나 업데이트할때
사용하며 데이터를 Body에 담아 보내므로 GET요청 보다는 보안이 상대적으로 높다는 것입니다.

추가적으로 GET 요청은 멱등이며 POST는 멱등이 아닙니다.
```

Q. 멱등이란?

```
멱등의 사전적 정의는 연산을 여러 번 적용하더라도 결과가 달라지지 않는 성질을 의미한다.

GET은 리소스를 조회한다는 점에서 여러 번 요청하더라도 응답이 똑같을 것 이다. 반대로 POST는 리소스를 새로 생성하거나 업데이트할 때 사용되기 때문에 멱등이 아니라고 볼 수 있다. (POST 요청이 발생하면 서버가 변경될 수 있다.)
```

Q. GET과 POST 이외의 HTTP 메서드에 대해

```
GET과 POST 이외에도 HTTP 메서드에는 PUT, DELETE, PATCH, HEAD, OPTIONS 등이 있습니다. 각각의 메서드는 다양한 목적과 의미를 가지고 있으며, 리소스의 생성, 업데이트, 삭제, 헤더 정보 조회 등의 다양한 작업을 수행할 수 있습니다.
```

Q. POST 요청에서 요청헤더의 Content-Type을 설정해야 하는 이유는 무엇?

```
서버가 요청을 올바르게 처리하고 해석할 수 있도록 도와주기 위함입니다. 서버는 헤더를 확인하여 요청데이터를 적절히 해석하고 처리할 수 있는데
예를들어 요청 데이터가 JSON 형식이라면 Content-Type을 "application/json"으로 설정해야 서버가 해당데이터를 JSON으로 인식하여 처리할 수 있습니다.

이처럼 Content-Type을 올바르게 설정하지 않는다면 서버는 요청 데이터를 올바르게 해석하지 못하거나 잘못된 형식으로 처리할 수 있으므로 Content-Type을 설정하여
서버가 요청을 올바르게 이해하고 처리할수 있도록 하기위해 Content-Type을 설정해주어야 합니다.
```
