Q. async / await에 대해 설명해보세요.

```
async/await은 JavaScript의 비동기 처리 패턴 중에서도 최근에 등장한 문법입니다. 이 문법은 기존의 비동기 처리 방식인 콜백 함수나 프로미스의 단점을 보완하여 코드의 가독성을 높일 수 있도록 도와줍니다.

async 키워드는 함수를 선언할 때 해당 함수가 비동기적인 동작을 포함하고 있음을 나타냅니다. 함수 앞에 async를 붙이면 해당 함수는 프로미스를 반환합니다. 이렇게 반환된 프로미스는 비동기 작업의 결과를 나타내는 값이 됩니다.

await 키워드는 async 함수 내에서 프로미스를 반환하는 코드 앞에 사용할 수 있습니다. await을 특정 코드 앞에 붙이면 해당 프로미스가 처리될 때까지 기다렸다가 다음 작업을 수행할 수 있습니다. await은 프로미스가 처리되고 값을 반환할 때까지 실행 흐름을 일시적으로 정지시키는 역할을 합니다.

일반적으로 await 키워드는 프로미스를 반환하는 API 호출과 같은 비동기 작업에서 사용됩니다. await을 사용하면 API에서 받아온 데이터를 특정 변수에 담아서 사용 가능할 때까지 기다린 후, 사용 가능한 상태가 되었을 때 다음 코드를 실행합니다.

또한, async/await은 try-catch 문을 사용한 예외 처리도 간편하게 할 수 있습니다. try 문 안에는 정상적으로 실행할 코드를 작성하고, 발생한 오류는 catch 문으로 잡아내어 처리합니다. 오류는 error 객체에 담겨지기 때문에 에러의 유형에 맞게 적절한 예외 처리를 수행할 수 있습니다.

이러한 방식으로 async/await을 사용하면 비동기 코드를 직관적이고 가독성이 좋게 작성할 수 있습니다. async/await은 코드의 구조를 단순화하고 오류 처리를 간편하게 만들어주는 장점을 가지고 있습니다.
```

Q. asycn await이 콜백함수나 프로미스의 어떠한 단점을 보완했는가?

```
우선 콜백지옥을 해결해줍니다. 콜백 함수의 경우 중첩하여 사용하게 되면 코드가 복잡해지는 콜백 지옥 문제를 만나게 되는데
async await을 사용하면 비동기 작업을 동기적인 코드처럼 표현이 가능해 이러한 문제를 해결할 수 있도록 도와줍니다.

또한 에러 처리를 간편하게 해줍니다. 기존의 비동기 처리 방식에서는 에러 처리를 위해 ".catch()" 메서드를 사용해야 했습니다.
catch메서드를 여러번 사용하다보면 에러 핸들링 코드가 중복될 우려가 있습니다.
그러나 async await에서는 try-catch 블록을 사용하여 에러 처리를 직관적이고 간편하게 할 수 있습니다.

이렇듯 async await은 기존의 비동기 처리방식의 단점을 보완하고 비동기 코드를 직렬적으로 작성할 수 있으므로 코드의 유지 보수성이 향상되고
코드의 실행 순서를 명확하게 파악할 수 있다는 장점이 있습니다.
```

Q. 그렇다면 Promise에서는 코드의 직렬화가 불가능한가?

```
Promise에서도 코드의 직렬화가 가능합니다. 프로미스 체이닝을 사용하면 비동기 작업을 직렬적으로 작성하는 것이 가능해지는데
이는 .then() 을 사용하여 이전 Promise가 처리되고 결과를 반환할 때 까지 기다리고 그 후에 다시 .then()을 이어붙여 직렬화가 가능합니다.

하지만 async await을 사용하게 되면 이러한 프로미스 체이닝이 없이도 코드의 직렬화가 가능하므로 코드의 가독성을 향상시키고
디버깅을 용이하게 만들어줍니다.

따라서 Promise에서도 코드의 직렬화는 가능하지만 async await을 사용하면 프로미스 체이닝없이 코드의 직렬화가 가능하므로 더 간편하다고 볼 수 있습니다.
```
