Q. CORS에러에 대해 설명해주세요.

```
웹 브라우저는 동일 출처 정책에 따라 웹페이지가 로드된 도메인과 다른출처의 리소스 요청을 제한합니다.
CORS는 이러한 제약을 허용적으로 변경하여 다른 출처의 리소스에 접근할 수 있도록 허용하는 메커니즘입니다.
여기서 출처가 다른 리소스란 프로토콜, 도메인, 포트번호중 하나라도 맞지 않다면 다른 출처로 인식하게 되고
CORS 설정을 해주지 않았을 경우 CORS에러를 발생시킵니다.

다른 도메인의 리소스에 접근해야 할 일이 생긴다면 해당 도메인의 서버는 CORS 정책을 설정하여 리소스 공유를 허용해야 합니다. 이를 위해서는 서버가 응답에 Access-Control-Allow-Origin 헤더를 포함하여 허용되는 도메인을 명시해야 합니다.

따라서, CORS 에러가 발생한다면 보통은 서버의 CORS 설정이 필요한 경우입니다. 서버 측에서 올바르게 CORS 정책을 설정하거나, 프록시 서버를 사용하여 CORS 이슈를 해결할 수 있습니다.
```
