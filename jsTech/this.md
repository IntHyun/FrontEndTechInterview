Q. this에 대해 설명해보세요.

```
this는 함수가 호출될 때 결정이 되는 동적인 값이며 호출 방법에 따라 this의 값은 달라질 수 있습니다.

먼저 this를 전역 범위에서 사용하게 되면 전역 객체인 window를 참조합니다.

그리고 함수 내부에서 this를 사용할 때는 호출 방법에 따라 달라집니다.

먼저 일반적인 방법으로 함수내부에서 호출될 때에는 this는 전역 객체를 참조합니다.
그리고 객체의 메서드로 함수가 호출될 때 this는 해당 객체를 참조하게 됩니다.
생성자 함수로 객체를 생성할때의 this는 생성되는 새로운 객체를 참조하게 됩니다.
화살표 함수의 this는 화살표 함수가 정의된 위치에서 상위 스코프의 this를 참조하게 됩니다.

이러한 this를 bind, call, apply와 같은 메서드로 this의 값을 고정시켜줄수도 있습니다.

먼저 call은 첫번째 인수로 전달할 인수를 전달합니다. 그 뒤 두번쨰 인수에 수동으로 지정해준 객체를 적어주게 되면 this는 수동으로 지정해준 객체를 참조하게 됩니다.
그러나 call은 함수에서 전달할 인수가 여러개라면 인수를 하나하나 전달해주어야 하는 불편함이 있습니다.

이때 apply를 사용하여 인수를 배열로 보내줄 수 있습니다. apply는 인수를 배열로 보낼 수 있어 두개의 인수만을 사용하는데 이외에는 call과 작동방식은 동일합니다.

bind의 경우 this의 값을 어디서 사용하든 호출 객체가 바뀌지 않게 고정시켜 버립니다. 마찬가지로 bind의 첫번째 인수에 고정할 객체를 선택하여주면
앞으로 해당함수는 어떤 경우에서든 고정시켜준 객체를 참조하게 됩니다.

이렇게 call, bind, apply를 사용하여 this의 값을 고정시켜 줄 수 있지만 예외적으로 화살표 함수 내부에서의 this는 동적으로 바인딩 되지않고
언제나 상위 스코프의 this를 가리키는 것으로 고정되기 떄문에 call, bind, apply와 같은 메서드를 사용하여 this를 변경할 수 없습니다.
```

Q. 굳이 this를 사용하는 이유가 무엇일까요?

```
myDiner라는 객체안에 name이 김치찌개이고 menu라는 함수가 정의되어 있다고 가정하겠습니다. 이 함수는 콘솔창에 "오늘 저녁은" myDiner.name을 출력해주는 함수라고 가정하겠습니다.
이때 menu함수는 myDiner 변수의 이름이 수정이 되거나 함수 자체를 다른 함수에서 사용하고 싶을 때 사용이 불편할 수 있습니다.
예를들어 myDiner의 변수이름이 yourDiner라고 바뀌게 된다면 menu함수에서 출력해주는 값 또한 수정해주어야 하는 불편함을 겪을 수 있습니다.
또한 menu라는 함수를 떼서 다른 객체에서 사용하게 된다고 하더라도 그 객체가 myDiner라는 변수를 가지고 있을지도 모르는 일입니다.

이러한 경우에 myDiner.name을 출력해주는 것 대신 this.name으로 출력해주는 방식으로 바꾸게 된다면 이때의 this는 해당 객체를 참조하게 되므로
변수의 이름이 바뀌어도 계속해서 값은 해당 객체를 참조하게 되고 다른 객체에서 사용한다 할지라도 그 객체를 참조하므로 좀 더 유동적인 대응이 가능해집니다.
```

Q. call() 과 apply() 의 차이에 대해 알고계신가요?

```
call은 함수를 실행 할 때 전달할 인수를 하나 하나 전달한다면 apply는 전달할 인수를 배열로 묶어 한번에 전달합니다.
그래서 인수를 두개만 사용합니다.
인수를 배열로 보낸다는 점이 두 메서드의 차이점이고 그외에는 동일한 기능을 수행합니다.
```

Q. call, apply, bind 모두 지정한 객체를 참조한다는게 똑같은데 어떻게 차이가 나는가?

```
사용 방법에 따른 차이라고 말씀드릴 수 있을 것 같습니다. call과 apply는 함수를 호출하면서 실행이 됩니다.
하지만 bind의 경우 새로운 함수를 반환하여 나중에 실행할 때 사용한다는 차이점이 있습니다.
```
