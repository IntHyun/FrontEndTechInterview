Q. SSR과 CSR에 대해 설명해주세요

```
클라이언트 사이드 렌더링(CSR)은 최초 접속 시 한번에 페이지 전체를 로드하고 이후부터는 특정 부분만 비동기 통신을 이용해서 데이터를 바인딩하는 방식으로 SPA에 적합한 환경입니다.
하나의 HTML을 두고 JavaScript를 이용하여 DOM을 동적으로 조작하여 필요한 부분만 렌더링 하기 때문에 화면 깜빡임 현상이 없고 UX 관점에서 성능이 좋습니다.
하지만 필요한 정보를 처음에 모두 받아오고 스크립트 파일에 의존적이기 때문에 최초 로드 시 구동 속도가 SSR 보다 느리고 SEO 대응이 번거롭다는 단점이 있습니다.

서버 사이드 렌더링(SSR)은 서버에서 완전하게 만들어진 HTML 파일을 받아와서 페이지를 렌더링하고 클릭과 같은 사용자의 인터랙션이 발생할 때마다 상황에 맞는 HTML을 받아와서 렌더링 하는 방식으로 MPA 구동 방식입니다.
CSR보다 최초 로딩 속도가 빠르고 웹 크롤러의 크롤링이 수월하여 SEO 대응에 적합합니다.
하지만 요청이 들어올 때 마다 페이지를 새로고침 하기 때문에 화면 깜빡임 현상이 발생하고 각 페이지별 전환 속도는 CSR보다 느리고 반응형 웹으로는 유저 친화적이지 않다는 단점이 있습니다.
```
