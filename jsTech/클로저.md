Q. 클로저에 대해 설명해달라

```
클로저는 특정함수에서 내부함수가 존재하고 이 내부함수가 외부함수가 리턴되었음에도 변수를 참조하고 있을 때 생기는 현상을 말합니다.

자바스크립트는 내부 함수에서 자신을 포함하는 외부 함수의 스코프에 접근할 수 있는데 내부 함수가 살아있는 상태에서 외부 함수가 return되면
외부 함수의 변수들에 대한 접근 권한은 내부 함수만이 가질 수 있게 됩니다. 이때 내부함수를 클로저 함수라 합니다.
이러한 특징을 이용한다면 변수를 외부로부터 은닉하고 함수 내부에서만 접근이 가능하므로 변수의 접근을 제어하고 정보를 보호할 수 있습니다.

그러나 클로저 함수가 외부 함수의 변수를 계속 참조하게 되면 비공개 프로퍼티가 생성되는 것이라서 가비지 컬렉터가 이를 인식하지 못하고 해당 변수가 메모리에서 해제되지 않고 계속 유지되면서 메모리 누수가 발생할 수 있습니다.

이러한 메모리 누수를 막기위해 필요한 경우에만 클로저를 사용하고 더 이상 필요하지 않은 경우에는 참조를 해제해야 합니다.
이를 위해 클로저를 사용한 후에 더이상 해당 변수를 참조하지 않는다면 해당 변수에 null 값을 할당하여 참조를 해제하는 방법을 사용할 수 있습니다.

정리하자면 클로저 공간에 대한 접근 권한을 가진 함수가 클로저이며 적절한 사용과 메모리 관리에 주의하면서 사용하도록 하는것이 중요합니다.
```

Q. 가비지 컬렉터가 무엇인가?

```
가비지 컬렉터는 JS에서 메모리 관리를 자동으로 처리하는 프로세스입니다.
가비지 컬렉터는 사용되지 않는 객체를 감지하고 있다가 메모리를 회수하는 방법을 사용합니다.

가비지 컬렉터 덕분에 메모리 누수를 최소화 할 수 있고 개발자는 일반적으로 직접적인 메모리 관리를 신경쓰지 않아도 됩니다.

하지만 가비지 컬렉션의 성능에는 약간의 차이가 있을 수 있으므로 큰 메모리 사용량이나 실행 속도에 민감한 앱에서는
가비지 컬렉션의 동작 방식을 이해하는 것이 중요합니다.
```
