Q. props와 state의 차이에 대해 설명하세요.

```
props와 state는 둘 다 컴포넌트에서 데이터를 다루는 데 사용되지만 약간의 차이점이 존재합니다.

먼저 props는 부모 컴포넌트에서 자식 컴포넌트로 전달되는 데이터입니다.
자식 컴포넌트에서는 전달된 props를 그대로 사용하며 자식 컴포넌트에서의 props는 변경이 불가능합니다.

state는 컴포넌트 내부에서 정의되고 사용되는 데이터입니다. state는 props와 다르게
내부 컴포넌트에서 자유롭게 변경이 가능하며 내부 동작과 렌더링에 영향을 줍니다.

주요 차이점을 정리하면 내부에서 정의되는 데이터냐 외부에서 정의되는 데이터냐에 따른 차이점과
불변성을 가지냐 변경이 가능한 데이터냐에 따른 차이점이 있을 수 있겠습니다.
```

Q. state를 직접 변경하지 않고 setState를 사용하는 이유?

```
우선 React는 가상돔을 사용하여 컴포넌트를 업데이트 합니다. setState를 사용하면
React가 필요한 경우에만 실제 돔을 업데이트 하도록 선택적으로 렌더링을 수행할 수 있습니다.
또한 여러 setState호출을 하나의 업데이트로 배치 처리하여 성능을 향상시키기 떄문입니다.

또다른 이유로는 state는 불변성을 유지해야하기 떄문입니다. setState를 사용하여 상태를 변경하면 React는
이전 상태와 새로운 사앹를 비교하여 변경사항을 찾고 렌더링을 수행합니다. 그러나 state를 직접 변경해주게 되면
리액트에서는 상태 변화가 발생하지 않았다고 가정하여 렌더링을 수행하지 않습니다.

이러한 이유들 떄문에 state는 직접 변경하기보다 setState를 통해 관리해주는 것입니다.
```
