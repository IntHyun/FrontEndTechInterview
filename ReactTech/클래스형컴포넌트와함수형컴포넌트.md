Q. 클래스형 컴포넌트와 함수형 컴포넌트의 차이

```
먼저 문법적으로 살펴보면 클래스형 컴포넌트는 ES6의 클래스 문법을 사용하여 정의됩니다.
반면에 함수형 컴포넌트는 JS의 함수형태로 정의되며 함수 본문 내에서 JSX를 반환하여 컴포넌트를 정의합니다.

또한 클래스형 컴포넌트는 state를 사용하여 내부 상태를 관리할 수 있는데 state는 클래스의 속성으로 정의되고 setStatee 메서드를 통해
업데이트 됩니다. 함수형 컴포넌트에서는 이전에는 상태관리를 위해 state를 사용할 수 없었지만 훅이 도입된 이후 useState훅을 사용하여
상태를 관리할 수 있게 되었습니다.

클래스형 컴포넌트는 생명주기 메서드를 사용하여 컴포넌트의 생명주기를 관리합니다. 함수형 컴포넌트에서는 이전에는 생명주기 메서드를 사용할 수 없었지만
리액트 훅을 통해 useEffect 훅을 사용하여 생명주기와 관련된 동작을 수행할 수 있게되었습니다.

최근에는 함수형 컴포넌트와 훅의 사용이 권장되며 함수형 컴포넌트가 주로 사용되는 추세입니다.
함수형 컴포넌트는 코드의 간결성과 재사용성 측면에서 이점을 가지며, 리액트 훅을 통해 클래형 컴포넌트에서 제공되던 기능을 대부분 사용할 수 있게 되었습니다.
따라서 함수형 컴포넌트를 사용하는 것이 권장되며 팀의 개발 스타일에 따라 클래스형 컴포넌트를 사용하는 경우도 있을 수 있습니다.
```
